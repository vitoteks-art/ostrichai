# Twitter/X Integration Setup Guide

This guide provides step-by-step instructions to set up the Twitter/X API integration for OstrichAi, enabling automated posting with media support.

## 1. Create a Twitter Developer Account

1.  Go to the [Twitter Developer Portal](https://developer.twitter.com/).
2.  Sign in with your Twitter/X account.
3.  Apply for a Developer Account if you haven't already (The **Free** or **Basic** tier is required for posting).

## 2. Create a Project and App

1.  In the Dashboard, click **"Create Project"**.
2.  Give your project a name (e.g., `OstrichAi`).
3.  Select your use case (e.g., `Building tools for businesses`).
4.  Provide a description for your project.
5.  Create a new App within the project.

## 3. Configure OAuth 2.0 Settings

1.  Navigate to your App's **"Settings"** tab.
2.  Click **"Set up"** under **"User authentication settings"**.
3.  Enable **OAuth 2.0**.
4.  Set **"Type of App"** to **"Web App, Automated App or Bot"**.
5.  **App Config**:
    *   **Callback URI / Redirect URL**:
        *   Development: `http://localhost:5173/oauth/callback/twitter`
        *   Production: `https://yourdomain.com/oauth/callback/twitter`
    *   **Website URL**: `https://yourdomain.com`
6.  Click **"Save"**.

## 4. Required Scopes

Ensure your app has the following scopes enabled in the OAuth 2.0 settings:
*   `tweet.read`
*   `tweet.write`
*   `users.read`
*   `offline.access` (Required for refresh tokens)

## 5. Obtain API Credentials

1.  Go to the **"Keys and Tokens"** tab.
2.  Under **"OAuth 2.0 Client ID and Client Secret"**, click **"Generate"** (or "Regenerate").
3.  **IMPORTANT**: Copy the **Client ID** and **Client Secret** immediately. You will not be able to see the secret again.

## 6. Configure Environment Variables

### Frontend (.env)
Add these to your local `.env` file:
```env
VITE_TWITTER_CLIENT_ID=your_client_id_here
```

### Supabase Edge Functions (Secrets)
Deploy the secrets to Supabase so the Edge Function can access them:
```bash
supabase secrets set TWITTER_CLIENT_ID=your_client_id_here
supabase secrets set TWITTER_CLIENT_SECRET=your_client_secret_here
```

## 7. Media Upload Note

The integration uses Twitter API v1.1 for media uploads and v2 for posting tweets. Ensure your App permissions allow for both (Standard v2 access usually includes this).

## 8. Testing the Integration

1.  Start your development server: `npm run dev`.
2.  Navigate to **Settings -> Connected Accounts**.
3.  Click **"Connect"** next to Twitter/X.
4.  Authorize the app on Twitter.
5.  Verify the account appears as "Connected".
6.  Go to the **Social Media Post Generator**, create a post with an image, and try posting it to your Twitter account.

---

> [!TIP]
> If you encounter `redirect_uri_mismatch`, double-check that the Callback URI in the Twitter Developer Portal exactly matches the one generated by your app.
